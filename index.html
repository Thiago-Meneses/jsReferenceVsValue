Object Reference vs Values

Passing by Value

1º Step:

Declaring two variables 

                            
let a = 10
let b = "Hello"

Both variables were passed by value

Ok, but what does Passing by value mean?

It means that these variables were directly linked to some value

There is no intermediary step from the variables that are being declared to the the values which are being assigned to them.

In our example:

The values on the right ( 10 and Hello ) will be directly assigned to the variables on the left ( a and b )

a is directly linked to the value 10

b is directly linked to the value "Hello"

Variables are only passed by value when we assign Primitive Types to them

2º Step:

Declaring another variable

                            
let a = 10
let b = "Hello"
let c = a

The newly created variable c is assigned with value a

The value on the right (a) will be directly assigned to the variable on the left (c)

The value of c will be equal to 10

3º Step:

                            
let a = 10
let b = "Hello"
let c = a
let c = c + 1

We add another step to show that our variables are indeed being passed by value, we add 1 to c

c is now equal to 11

a and b remain unchanged

When we assigned the value of a to c, both variables became equal to 10.

When we changed c, the variable a was not altered

No matter what we do with c, a remains unaffected, and vice-versa.

These variables, even if equal in value, were assigned to two "different" number tens, independent of eachother.

Both were equal to 10, but these weren't the same number 10

Passing by Reference

Now we are going to see how passing by reference is completely opposite to what we've just learned.

1º Step:

Declaring three variables 

                            
let a = 10
let b = "Hello"
let c = [1, 2]

When talking about Passing by Reference, we need to include Addresses and the Value which they contain.

We declared the same previous variables, but c now has been assigned with an array

This opens up to a new information we have to keep in mind

Arrays, Objects and essentially anything that is not a Primitive Type, will be Passed by Reference

This is why variable c, instead of having the expected value of [1, 2] , actually has a value of 00x01

This value is just a Memory Address

More specifically, the value 0x01 is an example for what a Memory Address name might look like.

This Memory Address is what actually holds the real value inside the memory of your computer

Variable c just stores a reference to that memory address

The memory address itself is what actually has the assigned value

2º Step:

Declaring a new variable

                            
let a = 10
let b = "Hello"
let c = [1, 2]
let d = c

We have a new variable d assigned with c as value

As we saw previously, c stores a link to a memory address, and this address is what is going to be assigned to d, not the actual array itself.

c and d are now referencing the same Memory Address

3º Step:

                            
let a = 10
let b = "Hello"
let c = [1, 2]
let d = c
d.push(3)

After pushing 3 to the variable d, the value this Memory Address refers has been altered.

The value from the Memory Address has been altered and no longer is [1, 2], but instead [1, 2, 3]

If both c and d are referencing the same memory address, something is going to happen.

Even if we didn't want to, c is also going to change

As we have changed the value itself, and not the reference to the value, both variables will be assigned with a newly changed value

We have modified two variables when we only wanted to modify one
